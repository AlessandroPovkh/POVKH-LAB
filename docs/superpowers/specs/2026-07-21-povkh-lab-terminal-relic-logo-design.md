# POVKH LAB — Terminal Relic logo concept

Date: 2026-07-21
Status: Approved master direction; production rollout authorized

## Approval record

On 2026-07-21 the user approved the Terminal Relic concept board as the new
POVKH LAB master direction and explicitly authorized its rollout to the brand
book, website and GitHub Pages. The production system must preserve the exact
supplied ornament geometry and the no-targeting exclusions below.

## Goal

Develop an adaptive POVKH LAB logo concept from the supplied `❧` vector. The direction combines an organic relic-like mark with disciplined HUD/terminal typography and ASCII language. It must feel like an underground electronic-music operating system, not decorative science fiction.

## Relationship to the current identity

Terminal Relic supersedes `Concept 02 / Typographic Monolith` as the normative
master. Concept 02 remains a process archive and must not be used for new
production work. Existing public asset filenames stay stable while their
geometry is replaced by Terminal Relic compositions.

Recognizable POVKH LAB codes remain intact:

- Void `#080808`, Bone `#F2EFE7`, Signal Red `#F32222`;
- Barlow Condensed for the name;
- IBM Plex Mono for terminal metadata;
- strict alignment, strong negative space and one signal accent.

## Core idea

The `❧` is treated as a recovered signal or living glyph inside a precise archival system. Its asymmetry and organic movement provide tension against the monolithic name and monospaced system labels.

HUD is expressed through information structure: indexes, file-like identifiers, status syntax, alignment rules and terminal spacing. It is not expressed through targeting graphics.

## Adaptive logo system

### 1. Full lockup

- simplified `❧` mark paired with `POVKH LAB`;
- clear silhouette and restrained spacing;
- optional secondary metadata line such as `POVKH_LAB::SIGNAL_001` for large applications;
- primary lockup remains legible without the metadata line.

### 2. Compact mark

- a carefully cleaned version of the supplied `❧` geometry;
- one-color by default;
- no surrounding frame or decorative HUD furniture;
- intended for avatar, favicon, watermark and small publisher mark after small-size validation.

### 3. ASCII signature

- a separate monospaced interpretation of the mark, not an automatic vector trace;
- accompanied by the exact brand name `POVKH LAB` or system identifier `POVKH_LAB::SIGNAL`;
- used in motion, terminal-like screens, metadata panels, inserts and process material;
- never substitutes for the primary lockup where immediate recognition is required.

## Explicit exclusions

- no crosshairs, reticles, targets or scope graphics;
- no circles or corner brackets that imply aiming;
- no glow, chrome, bevel, 3D, neon gradients or glossy cyberpunk effects;
- no random code, fake coordinates or meaningless technical decoration;
- no modification of the public brand name or catalogue-code rules;
- no unreviewed changes to the approved ornament geometry.

## First review artifact

Create one concept board showing the same direction in four controlled views:

1. full horizontal lockup;
2. compact mark;
3. ASCII signature;
4. a small-scale and monochrome test.

The board uses only the approved color tokens. It is a visual decision artifact, not a production export package.

## Evaluation criteria

- the mark remains recognizable at small size;
- `POVKH LAB` reads immediately;
- HUD character comes from structure rather than decoration;
- the organic symbol and rigid typography feel intentionally tense;
- the system works in pure black and white;
- the concept does not resemble a targeting interface;
- every downstream consumer uses the same governed production masters.
